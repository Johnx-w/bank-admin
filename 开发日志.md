## 一、技术选型

| 决策项 | 选择 | 原因 |
|--------|------|------|
| UI 框架 | React 19 | 生态最成熟，中后台社区方案丰富 |
| 构建工具 | Vite 8 | 比 CRA 快约 10x，原生 ESM，与 Vitest 共用配置 |
| 语言 | TypeScript 6 strict | 银行项目类型安全是硬要求 |
| 组件库 | Ant Design 6 | 中后台事实标准，ProTable 省大量模板代码 |
| 状态管理 | Zustand 5 | 比 Redux 轻约 90%，API 直觉化，TS 原生友好 |
| Mock 方案 | MSW 2 | Service Worker 层拦截，浏览器 DevTools 可见，无需额外进程 |
| 测试框架 | Vitest 4 | 与 Vite 共用配置，零额外配置成本 |
| 图表 | ECharts 6（按需引入） | 金融场景刚需，Tree-shaking 后体积仅 +80KB |
| HTTP 客户端 | Axios | 拦截器机制天然适合 Token 注入和统一错误处理 |
| Linter | ESLint | 技术成熟 |

脚手架命令：

```bash
npm create vite@latest bank-admin -- --template react-ts
```

## 二、开发规范

写功能代码之前先确立了 `instructions.md`，要点：

1. **TypeScript 严格模式** — 禁止 `any`，所有函数明确定义入参和出参类型
2. **命名规范** — camelCase / UPPER_SNAKE_CASE / PascalCase / kebab-case
3. **JSDoc 规范** — 所有导出函数标注功能、参数、返回值
4. **单一职责** — 函数不超过 50 行
5. **错误处理** — 异步操作必须 try-catch，向用户展示友好提示
6. **常量管理** — 魔法数字 / 枚举值集中在 `constants.ts`
7. **React 渲染优化** — useCallback 记忆化、条件渲染用三元而非 `&&`、early exit 模式

这份文档对后续所有批次开发起"宪法"作用，确保多人协作时质量不滑坡。

## 五、DevOps 与工程化

### 5.1 CI/CD — GitHub Actions

文件：`.github/workflows/ci.yml`

每次 push / PR 到 main 分支自动触发流水线：

1. Checkout → Setup Node 22
2. `npm ci` — 干净安装（锁定 lockfile）
3. `npm run lint` — Oxlint 代码风格
4. `npm run typecheck` — TypeScript 编译检查
5. `npm test` — 全部单测
6. `npm run build` — 生产构建

三道关卡通过才能合并 PR。踩坑：Windows runner 路径分隔符（`\`）vs Ubuntu runner（`/`），测试 snapshot 统一改用 POSIX 路径。

### 5.2 Docker 容器化

两次迭代。第一版 multi-stage（node build → nginx 托管静态文件）发现 MSW Mock 在纯静态模式下不工作。

最终双模式策略：

| 模式 | Dockerfile | 基础镜像 | 启动方式 | 产物大小 |
|------|-----------|---------|---------|---------|
| 开发 | `Dockerfile.dev` | node:22-alpine | `vite --host` | — |
| 生产 | `Dockerfile` | node:22-alpine → nginx:stable-alpine | nginx 托管 dist | ~20MB |

生产模式 multi-stage：

```dockerfile
# Stage 1: build
FROM node:22-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

# Stage 2: serve
FROM nginx:stable-alpine
COPY --from=builder /app/dist /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

docker-compose.yml：

```yaml
services:
  bank-admin:
    build: .
    ports:
      - "8080:80"
    restart: unless-stopped
```

踩坑：国内 Docker Hub 需配镜像加速器；PowerShell 写入的 Dockerfile 含 CRLF 换行导致部分指令解析异常，统一用 LF。

### 5.3 E2E 测试 — Playwright

选 Playwright 而非 Cypress：原生多浏览器支持、API 接近 Vitest、`webServer` 配置可直启 Vite。

```text
e2e/
├── playwright.config.ts
├── login.spec.ts          # 3 条用例
└── users.spec.ts          # 3 条用例
```

**用例覆盖：**
1. 正确凭据登录成功 → 跳转仪表盘
2. 错误凭据显示错误提示
3. 未登录访问受保护页面 → 重定向登录页
4. 用户列表正常加载 10 条数据
5. 关键词搜索过滤
6. 状态下拉筛选过滤

**核心踩坑：**

- **Zustand 状态丢失** — `page.goto('/users')` 导致页面完全重新加载，`authStore` 归零被 AuthGuard 拦截。修复：登录后用 React Router 客户端导航（点击菜单项）而非 `goto()`。
- **Ant Design Select 定位** — 虚拟列表渲染下 class 选择器不稳定。改用 `getByRole('combobox')` + 键盘操作（ArrowDown + Enter）。
- **placeholder 文本不匹配** — 搜索框实际用 `/` 分隔，测试写了中文顿号。

最终：6 passed (15.4s)。

## 六、踩坑汇总

| # | 坑 | 根因 | 修复 |
|---|-----|------|------|
| 1 | MSW v2 API 不兼容 v1 | `rest.get` → `http.get` | 全部按 v2 写法 |
| 2 | React 19 + Ant Design 6 类型不匹配 | 两者都是最新大版本 | 锁定 minor 版本 |
| 3 | TabsView setState during render | render 阶段禁止副作用 | 移入 useEffect |
| 4 | ECharts 全量引入 ~1MB | 未 tree-shaking | 按需引入，降至 ~80KB |
| 5 | 身份证校验不完整 | 缺校验位验证 | 补加权因子 + 模 11 映射（GB 11643） |
| 6 | PowerShell `Out-File -Encoding UTF8` 写 BOM | PowerShell 默认行为 | 改用 `[System.IO.File]::WriteAllText` 或 `fs.writeFileSync` |
| 7 | echarts-for-react CJS/ESM 不兼容 | `"type": "module"` 下 CJS 被误解析 | 自封装 ChartContainer |
| 8 | ErrorBoundary 路由切换不重置 | `hasError` 不随 `<Outlet />` 切换清除 | `getDerivedStateFromProps` 比较 `children.type` |
| 9 | Dockerfile CRLF 换行 | PowerShell 写入 Windows 换行 | 统一 LF |
| 10 | Docker Hub 不可达 | 国内网络 | 配镜像加速器 |
| 11 | E2E `goto()` 丢失 Zustand 状态 | in-memory store 随页面重载归零 | 改用 React Router 客户端导航 |
| 12 | Playwright 定位 Ant Design Select 不准 | 虚拟列表渲染 | `getByRole('combobox')` + 键盘操作 |
